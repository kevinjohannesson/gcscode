# Map (demo) — throwaway maplibre scaffold

**Status:** Approved (2026-05-03)

## Context

The roadmap's "Map" entry under Feature extensions → Coming is a real iteration that needs to design (a) maplibre integration, (b) a cross-extension API surface so other extensions can contribute layers / markers / overlays, and (c) operator-focused UX (full-bleed layout, follow modes, picking, selection state). That iteration is genuinely architecturally interesting and not small.

This spec is a **precursor scaffold**, not the real Map iteration. A new throwaway extension `@gcscode/extension-map-demo` proves out maplibre integration + the consumer-side cross-extension pattern (the demo reads SITL exports and renders a drone marker via the existing `getExtension` mechanism — no new public API surface). When the real Map iteration ships, the demo is removed.

The throwaway framing is deliberate: it lets the smallest cut land first (a working maplibre canvas in the running app, with a live drone marker from real SITL telemetry) without committing to any of the real map's API decisions. The real iteration then designs against knowledge gained from the demo.

## Goals

- New workspace package `@gcscode/extension-map-demo` mirroring the shape of `@gcscode/extension-vehicle-status` (private, `workspace:*` deps, type-only sibling import to `@gcscode/extension-sitl`).
- Adds `maplibre-gl` as a runtime dependency on the map-demo package only.
- Registers one view contribution (`gcscode.map-demo.main`) rendering a maplibre `Map` instance that fills its slot at fixed `400px` height.
- Reads `host.extensions.getExtension<SitlExports>('gcscode.sitl')?.exports.telemetry` reactively. When `lat` and `lng` are non-null, places a default maplibre `Marker` and centers the map on the same coordinates via `setCenter` (instant — no easing animation, since GLOBAL_POSITION_INT cadence at ~5 Hz would stack `easeTo` animations and feel laggy).
- Initial camera (pre-fix fallback): `[lng=149.17, lat=-35.36]`, zoom 13. Same coordinates the existing test data uses.
- **User pan / rotation disabled**, **zoom kept enabled.** Auto-follow as the only camera-position driver makes manual pan a UX dead-end (any pan reverts on the next telemetry tick). Disabling pan at construction time avoids the pan-fight entirely. Zoom stays on so the operator can adjust scale without interfering with the follow behavior. Specifically: `dragPan: false`, `boxZoom: false`, `dragRotate: false`, `keyboard: false` in the Map constructor, plus `map.touchZoomRotate.disableRotation()` post-construction to silence touch-rotation. `scrollZoom`, `doubleClickZoom`, and the zoom half of `touchZoomRotate` stay at their default-enabled state.
- Tile source: `https://demotiles.maplibre.org/style.json` — maplibre's official demo style. No API key, dev-permitted, vector tiles. Avoids OSM raster tile-usage-policy concerns + MapTiler-account setup. Acceptable cartographic detail for a throwaway demo.
- Adds the new extension to `bundledExtensions` (after sitl, since it depends on SITL exports — same insertion-order constraint as vehicle-status). Enabled by default; localStorage persistence carries through B4.
- One `Extension.deactivate?()` hook (second consumer after SITL): disposes the maplibre instance via `map.remove()` — releases the WebGL context.
- Roadmap update: new line under Feature extensions → Coming for `Map (demo)` flagged as throwaway scaffold; the real `Map` entry is unchanged.

## Non-goals

- **No cross-extension API on the map.** No `host.window.registerMapLayer`, no `host.map.*`, no marker contributions from other extensions. The demo is a self-contained consumer; the real Map iteration designs the API.
- **No follow-mode toggle.** Camera always tracks the marker (when fix exists). Manual pan is disabled at construction so there's no pan/follow conflict to manage; the real Map iteration tackles pan + follow-mode properly.
- **No heading-aware marker rotation.** Marker is just a default maplibre teardrop pin. Heading rendering is a real-map concern.
- **No selection state, picking, or click handlers.** Map is purely display.
- **No layout work.** The map renders inside the existing stacked view-contribution slot in `<section class="shell__content">`, sandwiched between the Example view (above) and the SITL location view. Operator who wants a focused map view disables the other extensions via the panel. The real Map iteration tackles full-bleed / split-pane / focal-point operator UX.
- **No SITL → vehicle rename.** The user's parallel observation about renaming `gcscode.sitl` → `gcscode.vehicle` (since "where the data comes from is irrelevant") is captured as a noted observation only; not addressed in this iteration. The map-demo `import type { SitlExports } from '@gcscode/extension-sitl'` exactly mirrors vehicle-status's pattern and will follow whatever rename happens later (one find/replace if/when it lands).
- **No ADR.** All decisions extend established patterns (workspace package shape per ADR-0001, view contribution per A-phase, type-only sibling import per ADR-0005, manifest per ADR-0007, deactivate hook per the deactivate-hook spec). No new architectural ground.
- **No error handling for tile-fetch failures.** maplibre handles its own errors; the demo doesn't surface them.
- **No tests of maplibre rendering.** maplibre is heavy to instantiate in jsdom and the demo is throwaway. Unit tests cover the extension's manifest, activation, and the SITL-exports read path. The browser smoke is the regression check.

### `packages/extension-map-demo/src/index.ts` (new)

```ts
import type { Extension, ExtensionHost } from '@gcscode/extension-api';
import type { SitlExports } from '@gcscode/extension-sitl';

import MapView from './map-view.svelte';

let host: ExtensionHost | null = null;

/**
 * Helper for the map view component to read SITL exports reactively.
 * Returns undefined if SITL is not currently activated. Reads inside a
 * `$derived` re-run when SITL enables / disables (registry's exports map is
 * a SvelteMap — see ADR-0005).
 */
export function getSitlExports(): SitlExports | undefined {
  return host?.extensions.getExtension<SitlExports>('gcscode.sitl')?.exports;
}

export const mapDemoExtension: Extension = {
  manifest: {
    id: 'gcscode.map-demo',
    displayName: 'Map (demo)',
    version: '0.0.0',
    description:
      'Throwaway scaffold: maplibre map with a drone marker driven by SITL telemetry. The real Map iteration replaces this.',
  },
  activate(context) {
    host = context.host;
    context.subscriptions.push(
      context.host.window.registerView({
        id: 'gcscode.map-demo.main',
        component: MapView,
      }),
    );
  },
  deactivate() {
    host = null;
  },
};
```

The maplibre `Map` instance lifecycle lives in the view component (created in `onMount`, destroyed in `onDestroy`), not in the extension's `deactivate?()`. This is correct because the view contribution's disposable unmounts the component when the extension disables; Svelte's onDestroy fires and disposes the maplibre instance. The extension's `deactivate?()` only clears the module-level `host` reference — symmetric with vehicle-status.

### `packages/extension-map-demo/src/map-view.svelte` (new)

```svelte
<script lang="ts">
  import 'maplibre-gl/dist/maplibre-gl.css';
  import maplibregl from 'maplibre-gl';
  import { onDestroy, onMount } from 'svelte';

  import { getSitlExports } from './index';

  let container: HTMLDivElement | undefined = $state();
  let map: maplibregl.Map | null = null;
  let marker: maplibregl.Marker | null = null;

  // Initial camera fallback before any fix arrives. Matches the SITL test data
  // (Canberra) so the demo has a sensible default location even when the
  // simulator isn't running.
  const INITIAL_CENTER: [number, number] = [149.17, -35.36];
  const INITIAL_ZOOM = 13;

  // Maplibre's official demo style. Free, no API key, dev-permitted vector
  // tiles. The real Map iteration may change tile sources.
  const TILE_STYLE = 'https://demotiles.maplibre.org/style.json';

  // Reactive read of SITL telemetry. When SITL is not active, exports is
  // undefined; when active but no fix yet, lat/lng are null.
  const telemetry = $derived(getSitlExports()?.telemetry);

  onMount(() => {
    if (!container) return;
    map = new maplibregl.Map({
      container,
      style: TILE_STYLE,
      center: INITIAL_CENTER,
      zoom: INITIAL_ZOOM,
      // Pan and rotation disabled — auto-follow is the only camera-position
      // driver, so manual pan would just fight the next telemetry tick. Zoom
      // stays enabled so the operator can adjust scale without interfering
      // with follow behavior.
      dragPan: false,
      boxZoom: false,
      dragRotate: false,
      keyboard: false,
    });
    // Touch zoom-rotate has a rotation half that's enabled by default; silence
    // it for consistency with dragRotate: false. Pinch-zoom (the zoom half)
    // stays enabled.
    map.touchZoomRotate.disableRotation();
  });

  onDestroy(() => {
    if (marker) marker.remove();
    if (map) map.remove();
    marker = null;
    map = null;
  });

  // Single effect: marker + camera both update from telemetry. setCenter is
  // instant (no easing) because GLOBAL_POSITION_INT cadence at ~5 Hz would
  // stack easeTo animations and feel laggy. Marker and camera tracking in
  // lockstep makes the visual smooth without animation.
  $effect(() => {
    if (!map) return;
    const t = telemetry;
    if (!t || t.lat === null || t.lng === null) {
      // No fix or SITL inactive — remove the marker if it was placed.
      if (marker) {
        marker.remove();
        marker = null;
      }
      return;
    }
    if (!marker) {
      marker = new maplibregl.Marker().setLngLat([t.lng, t.lat]).addTo(map);
    } else {
      marker.setLngLat([t.lng, t.lat]);
    }
    map.setCenter([t.lng, t.lat]);
  });
</script>

<div class="map-view">
  <h2 class="map-view__heading">Map (demo)</h2>
  <div bind:this={container} class="map-view__canvas"></div>
</div>

<style>
  .map-view {
    display: flex;
    flex-direction: column;
    gap: 0.5rem;
    padding: 0.5rem 1rem;
  }
  .map-view__heading {
    margin: 0;
    font-size: 1.125rem;
    font-weight: 600;
  }
  .map-view__canvas {
    width: 100%;
    height: 400px;
    border-radius: 4px;
    overflow: hidden;
  }
</style>
```

The `bind:this={container}` pattern uses Svelte 5's `$state` for the container ref. Maplibre needs a DOM element, so the `Map` instance is created in `onMount` after the div exists.

The CSS import (`maplibre-gl/dist/maplibre-gl.css`) is processed by Vite at build time and inlined into the page. No runtime cost beyond the bundle increment.

### `packages/extension-map-demo/src/index.test.ts` (new)

Tests follow vehicle-status's pattern: a fake `ExtensionHost` is constructed in-test (no shell-internals imports), the extension's `activate(context)` is called against it, and assertions read off the fake host's mock calls. This keeps the extension package self-contained for testing — no test-time dep on `@gcscode/shell`.

```ts
import { afterEach, describe, expect, it, vi } from 'vitest';

import type { Disposable, ExtensionHost, ViewContribution } from '@gcscode/extension-api';

import { getSitlExports, mapDemoExtension } from './index';

function makeFakeHost(opts?: {
  registerView?: ExtensionHost['window']['registerView'];
  getExtension?: ExtensionHost['extensions']['getExtension'];
}): ExtensionHost {
  return {
    window: {
      registerView: opts?.registerView ?? vi.fn(() => ({ dispose: () => {} }) as Disposable),
      registerStatusBarItem: vi.fn(() => ({ dispose: () => {} }) as Disposable),
      showQuickPick: vi.fn(),
    },
    commands: {
      registerCommand: vi.fn(() => ({ dispose: () => {} }) as Disposable),
      executeCommand: vi.fn(() => Promise.resolve()) as ExtensionHost['commands']['executeCommand'],
    },
    keybindings: {
      registerKeybinding: vi.fn(() => ({ dispose: () => {} }) as Disposable),
    },
    extensions: {
      getExtension: opts?.getExtension ?? vi.fn(() => undefined),
    },
  };
}

describe('mapDemoExtension', () => {
  afterEach(() => {
    // deactivate clears the module-level host reference; ensure clean slate
    mapDemoExtension.deactivate?.();
  });

  it('has the expected manifest', () => {
    expect(mapDemoExtension.manifest.id).toBe('gcscode.map-demo');
    expect(mapDemoExtension.manifest.displayName).toBe('Map (demo)');
    expect(mapDemoExtension.manifest.version).toBe('0.0.0');
    expect(mapDemoExtension.manifest.description).toContain('Throwaway scaffold');
  });

  it('activate registers exactly one view with id gcscode.map-demo.main', () => {
    const registerView = vi.fn((_view: ViewContribution): Disposable => ({ dispose: () => {} }));
    const host = makeFakeHost({ registerView });
    const subscriptions: Disposable[] = [];

    mapDemoExtension.activate({
      host,
      subscriptions,
      extension: { id: 'gcscode.map-demo', displayName: 'Map (demo)', version: '0.0.0' },
    });

    expect(registerView).toHaveBeenCalledTimes(1);
    expect(registerView).toHaveBeenCalledWith(
      expect.objectContaining({ id: 'gcscode.map-demo.main' }),
    );
    expect(subscriptions).toHaveLength(1);
  });

  it('getSitlExports returns SITL exports when host is active and SITL is registered', () => {
    const sitlExports = { telemetry: { lat: -35.36, lng: 149.17 } };
    const host = makeFakeHost({
      getExtension: vi.fn(() => ({ id: 'gcscode.sitl', exports: sitlExports })),
    });
    mapDemoExtension.activate({
      host,
      subscriptions: [],
      extension: { id: 'gcscode.map-demo', displayName: 'Map (demo)', version: '0.0.0' },
    });

    expect(getSitlExports()).toBe(sitlExports);
  });

  it('deactivate clears the module-level host so subsequent getSitlExports returns undefined', () => {
    const sitlExports = { telemetry: { lat: -35.36, lng: 149.17 } };
    const host = makeFakeHost({
      getExtension: vi.fn(() => ({ id: 'gcscode.sitl', exports: sitlExports })),
    });
    mapDemoExtension.activate({
      host,
      subscriptions: [],
      extension: { id: 'gcscode.map-demo', displayName: 'Map (demo)', version: '0.0.0' },
    });
    expect(getSitlExports()).toBe(sitlExports);

    mapDemoExtension.deactivate?.();

    expect(getSitlExports()).toBeUndefined();
  });
});
```

Maplibre is NOT instantiated in tests. The component itself isn't mounted — these are extension-shape tests covering the activate/deactivate contract and the helper. The browser smoke verifies the rendering.

### `packages/extension-map-demo/README.md` (new)

```md
# @gcscode/extension-map-demo

Throwaway scaffold demonstrating maplibre integration + the consumer-side cross-extension pattern. Reads SITL telemetry via `host.extensions.getExtension<SitlExports>('gcscode.sitl')` and renders a drone marker on a maplibre map. Camera tracks the marker.

The real Map iteration (roadmap "Coming → Map") replaces this once it lands. Don't depend on the demo for anything; the package will be removed.

## Contributions

- **View** — `gcscode.map-demo.main`, renders a 400px-tall maplibre canvas with a drone marker.

## Cross-extension dependency

- **`@gcscode/extension-sitl`** — type-only. Imports `SitlExports` via `import type`. Runtime lookup is via `host.extensions.getExtension('gcscode.sitl')` — no runtime coupling.

## Tile source

`https://demotiles.maplibre.org/style.json` — maplibre's official demo vector tiles. No API key required. Permitted for development / demos. Not suitable for production-tier use; the real Map iteration picks a production tile source.

## Behavior

- SITL not registered or disabled: no marker; camera stays at the initial fallback (Canberra, zoom 13).
- SITL active, no fix yet: no marker; camera unchanged.
- SITL active with `lat`/`lng`: marker placed at `[lng, lat]`; camera centered on the same coordinates instantly. Marker and camera update on every telemetry tick (~5 Hz from GLOBAL_POSITION_INT).
- User pan / rotation disabled. Wheel zoom, double-click zoom, pinch zoom (touch) stay enabled.

## Lifecycle

`deactivate()` clears the module-level `host` reference so subsequent `getSitlExports()` calls (e.g. during a re-enable race) return `undefined`. The maplibre `Map` instance is owned by the view component and disposed in its `onDestroy`.

- [ADR-0007](../../docs/decisions/ADR-0007-extension-manifest.md) — extension manifest
- [Spec 2026-05-03-map-demo](../../docs/specs/2026-05-03-map-demo.md) — this iteration
```
